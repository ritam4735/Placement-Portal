## Overview

The **Placement Portal** is a comprehensive system to manage the entire campus placement workflow. It connects three primary roles:
- **Students**: Can register, build their profiles, upload resumes, and apply for drives.
- **Companies**: Can register (requires admin approval), post placement drives, and manage applicants.
- **Administrators**: Moderate the system, manage master data (Courses, Departments, Batches, Semesters), and oversee the recruitment process.

## Installation & Setup

**1. Clone or download the repository**

Navigate to the project directory:
```bash
cd PlacementPortal
```

**2. (Recommended) Create and activate a virtual environment**

```bash
# Windows
python -m venv venv
venv\Scripts\activate

# macOS / Linux
python3 -m venv venv
source venv/bin/activate
```

**3. Install dependencies**

```bash
pip install -r requirements.txt
```

**4. Initialise the database**

This creates all tables (Users, Courses, Departments, Batches, Semesters, Student Profiles, Company Profiles, Placement Drives, Applications, and Testimonials) and seeds the default admin user. Run this once before starting the app.

```bash
python create_db.py
```

---

## Running the App

```bash
python app.py
```

The app will start at **http://127.0.0.1:5000**.

## Features by Role

### Admin
- **Dashboard**: Live statistics for students, companies, drives, applications, and placements.
- **Company Management**: Approve, reject, block, unblock, or delete company accounts.
- **Drive Moderation**: Approve or reject placement drives posted by companies.
- **Student Management**: Search, filter, and view full student profiles and application histories. Edit academic details and manage placement statuses.
- **Master Data**: Manage and configure Courses, Departments, Batches, and Semesters.
- **Testimonial Moderation**: Review and publish user testimonials for the landing page.

### Company
- **Registration**: Register as a company and await admin approval.
- **Drive Management**: Create new placement drives (specifying job title, description, eligibility, and deadline). Edit or delete drives while they are pending approval. Close drives when they are completed.
- **Applicant Tracking**: View the list of students who applied to approved drives and update their application status (e.g., Shortlisted, Selected, Rejected).

### Student
- **Registration**: Self-register and build a comprehensive academic profile.
- **Drive Participation**: View open and approved placement drives. Apply to eligible drives (duplicate applications are automatically blocked).
- **Profile Management**: Upload and update a PDF resume (max 2 MB), and edit personal information (name, email, phone, age, CGPA).
- **Real-time Tracking**: Monitor application statuses in real time.

## Database Schema

The database consists of 10 primary tables designed to track users, master data, profiles, and operations:
1. `users` - Core authentication and role tracking.
2. `courses`, `departments`, `batches`, `semesters` - Master tables for academic data.
3. `student_profiles`, `company_profiles` - Extended profiles linked to users.
4. `placement_drives` - Job postings by companies.
5. `applications` - Student applications to drives.
6. `testimonials` - User feedback displayed on the public landing page.

*(Refer to `dbSchema.pdf` for a visual ER diagram of the relationships.)*

---

## Notes

- The database (`portal.db`) is constructed programmatically using SQLAlchemy. Do **not** create or modify it manually via external tools like DB Browser, as it may break relationships.
- Resume files are saved in the `uploads/` folder, prefixed with the student ID to guarantee uniqueness.
- The application logic relies entirely on server-side rendering (SSR) using Jinja2 templates; JavaScript is used minimally and is not required for core functionality.
- To reset the database and start fresh, you can delete `portal.db` and re-run `python create_db.py`. This drops all existing data and creates empty tables with a default admin user.
